# DECISIONS.md — Decision Log (append-only)

2026-06-25 — Hero scene → "Live Annotation Feed"; scroll without drei ScrollControls:
Reason: Replaced the floating-monitor scene with a procedural annotation-session
scene (floor grid + fog + emissive game-world boxes + cyan wireframe annotation
boxes with drei <Html> labels + sweeping scan line + HTML HUD). The brief asked
for drei `useScroll`, but that requires <ScrollControls>, which creates its own
scroll container and hijacks the wheel over the canvas — incompatible with the
site's full-page Lenis smooth scroll (the hero is one section in a longer page).
Instead the scroll interaction (camera z 8→3, extra boxes, faster scan) is driven
by the hero section's page-scroll progress via a ref read in useFrame — same
effect, no scroll hijack. Bloom/post-processing removed entirely (it caused
flashing); glow now comes from emissive materials. @react-three/postprocessing
is left installed but unused.

2026-06-25 — Manual Vite scaffold instead of `npm create vite`:
Reason: The project folder already contained CLAUDE.md, which makes the
interactive scaffolder prompt ("directory not empty — remove files?"). Writing
package.json / configs by hand avoids the prompt and preserves the spec file.

2026-06-25 — Pinned Tailwind CSS v3.4 (not v4):
Reason: CLAUDE.md Part 3 assumes the `tailwind.config.js` + `postcss` +
`autoprefixer` + `npx tailwindcss init -p` workflow, which is the v3 model. v4
changed to `@tailwindcss/postcss` and a CSS-first config — pinning v3 keeps the
spec's design-token config valid and avoids breakage.

2026-06-25 — Pinned @react-three/fiber v8 + drei v9 + postprocessing v2:
Reason: R3F v9 / drei v10 require React 19. The spec targets React 18, so the
v8/v9/v2 line is the compatible, battle-tested set for React 18.

2026-06-25 — Annotation Canvas rendered via live CanvasTexture (Part 6.4):
Reason: The signature "boxes label themselves in real time" effect is drawn to
a 2D <canvas> and fed to Three.js as a CanvasTexture, redrawn each frame. This
needs no image/texture assets, stays crisp at any size, and makes the labels
trivially data-driven.

2026-06-25 — Hero 3D scene lazy-loaded with React.lazy (Part 10):
Reason: Three.js is ~890 kB. Code-splitting it into its own chunk keeps the
initial bundle small so first paint and the text content aren't blocked by WebGL.

2026-06-25 — Reduced-motion handled at the source, not just CSS:
Reason: Lenis, the R3F camera rig, the custom cursor, and every GSAP timeline
check `prefers-reduced-motion` and either disable or snap to final state, so the
site is fully usable and static for motion-sensitive users (Part 6.5).

2026-06-25 — Resume PDF generated by a dependency-free Node script:
Reason: `scripts/gen-resume.cjs` hand-builds a valid PDF with correct xref byte
offsets — a real downloadable placeholder with locked Part 5 content, no
headless-browser or library needed. Replace with a real export later.

2026-06-25 — OG image shipped as SVG (placeholder):
Reason: No raster tooling available in-session. SVG avoids a 404 and looks
correct; DEPLOY.md flags replacing it with a 1200×630 PNG for social previews.
